Таблицы в БД:

Users:
- Id (id пользователя)
- Username (юзернейм, одновременно и логин для входа. Не больше 20 символов)
— Role (флаги с правами)

Credentials:
- Id (id пользователя)
- Password (зашифрованный пароль)

Invites: (приглашения)
- Id
- OwnerId (id создателя)
- CreationTime (время создания)
- Code (12-значный набор букв и цифр)
— InvitedUserId (id пользователя, который был приглашён. Если 0 — то инвайт всё-ещё рабочий)
— InvitationTime (время приглашения)

Applications:
- Id (id приложения)
- OwnerId (id пользователя-создателя приложения)
— Name (имя приложения)
— PrivateKey (приватный ключ. Необходимо для расшифровки данных, которые само приложение (клиент) отправляет в Appman)

AppAccesses: (права доступа юзеров к приложению)
— Id
- ApplicationId (id приложения)
- UserId (id пользователя)
- ScopeFlags (флаги прав)

Branches: (ветки приложений)
- Id
- ApplicationId (id приложения)
- Name (название ветки)

AppBuilds: (бинарники приложения)
- Id
- ApplicationId (id приложения)
- Branch (название ветки приложения)
- BuildNumber (номер сборки. Например, если версия клиента выглядит как 1.19.153 — то должно быть указано 153)
— BinaryUrl (url бинарника)

CrashLogs: (отчёты об ошибках)
- Id
- ApplicationId (id приложения)
- Branch (название ветки приложения)
— Build (номер сборки приложения)
- OSName (название ОС (Windows, macOS, Android, и пр))
- OSVersion (версия ОС. Например, 10.0.19045 (для Windows), 14.0 (для Android))
— Timestamp (время отправки отчёта в unix-time)
- Code (код ошибки. Например, в винде это HResult)
- Info (сообщение об ошибке)
