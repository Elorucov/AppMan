Appman API:

Endpoint: <SERVER_DOMAIN>/appman/api/<method_name>

- auth.getAccessToken
Позволяет получить ключ доступа к Appman. 
* login (string) — юзернейм пользователя. Обязательное.
* password (string) — пароль. Обязательное.

Ответ: объект с полями ниже:
* userId (int) — ID пользователя
* accessToken (string) — ключ доступа
* expiresIn (int) — срок действия ключа


- users.get
Получает информацию о пользователе. (для теста)
* userId (int) — ID пользователя.

Ответ: объект с полями ниже:
* id (int) — ID пользователя (если не передан параметр userId, то ID текущего пользователя)
* username (string) — юзернейм пользователя.

- users.createInvite
Создаёт приглашение, с помощью которого можно зарегистрироваться в Appman. Доступен только для админа.

Ответ: объект с полями ниже:
* creationTime (long) — время создания приглашения.
* code (string) — код, который создатель должен отправить приглашаемому для регистрации в Appman.


- users.getInvites
Получает инвайты. Доступен только для админа.
* userId — id пользователя, созданные приглашения которого будет возвращены. Если не передан — то будет возвращены приглашения текущего пользователя.

Ответ: объект с полями ниже:
* count (int) — кол-во.
* items (array) — массив объектов со следующими полями:
  * id (int).
  * ownerId (int) — ID создателя приглашения.
  * creationTime (long) — время создания приглашения.
  * code (string) — код.
  * invitedUserId (int) — ID приглашённого пользователя. Если 0, значит, ещё не приглашён и код действительный.
  * invitationTime (long) — время, когда приглашённый пользователь зарегистрировался.
